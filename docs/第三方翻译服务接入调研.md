# 第三方翻译服务接入调研

## 文档状态

- 调研日期：2026-07-14
- 使用场景：个人自用的 Chrome Manifest V3 扩展，自动把 X 帖子翻译为简体中文。
- 本文阶段：只读调研与开发约束整理，尚未接入任何第三方翻译 API。
- 信息来源：各厂商官方产品、计费、API、错误码、安全和服务条款页面。
- 注意：免费额度、控制台入口和服务条款可能调整，正式开发和发布前应再次核对。

## 结论摘要

不能把六家厂商简单实现为“收到 429 就永久切到下一家”。429 通常只是短时限流；月免费额度耗尽、鉴权失败、欠费、参数错误必须分别处理。

| 厂商 | 当前免费能力 | 免费用完后的行为 | 能否可靠判断免费额度耗尽 | 当前扩展的主要约束 | 接入结论 |
| --- | --- | --- | --- | --- | --- |
| 腾讯云机器翻译 | 500 万字符/月 | 后付费默认关闭；额度耗尽自动停服 | 可以，`FailedOperation.NoFreeAmount` | 永久 SecretKey 在客户端有泄露风险；CORS 需实测 | 优先接入 |
| Azure Translator F0 | 200 万字符/月 | F0 耗尽停止，不自动升级计费 | 可以，`403001` | 每订阅只能有一个 F0；CORS 需实测 | 优先接入 |
| 阿里云机器翻译 | 100 万字符/月 | 自动进入后付费 | 不可以，没有免费额度耗尽错误 | 必须在本地约 98 万字符硬停；AccessKey 有泄露风险 | 可接入，但必须先做计费熔断 |
| 百度翻译高级版 | 100 万字符/月 | 超额计费，且计费/停服存在延迟 | 不可以 | 客户端不得缓存百度译文；必须本地约 95 万字符硬停 | 可接入，但需独立无持久缓存路径 |
| Google Cloud Translation | 前 50 万字符/月由每月 10 美元抵扣 | 超额自动计费 | 不可以 | 必须本地硬停；必须在译文旁署名；需启用 Billing | 可接入，但数据模型和 UI 必须支持来源署名 |
| DeepL | 旧 API Free：50 万/月；新 API Developer：总计 100 万、不会重置 | 达硬上限后停止 | 可以，HTTP `456`，也可查 `/usage` | API Free 已停止新购；官方禁止浏览器直连；Developer 不允许处理个人数据 | 不建议纳入普通 X 自动翻译链；仅保留已有旧账号或未来代理方案 |

按当前规则，建议默认顺序为：

1. X 自带翻译。
2. 腾讯云机器翻译。
3. Azure Translator F0。
4. 阿里云机器翻译。
5. 百度翻译高级版。
6. Google Cloud Translation Basic v2。
7. Chrome 本地 Translator API。

DeepL 不进入新用户默认链。只有用户已经拥有旧 API Free 账号，或以后增加合规代理并确认内容不含个人数据时，才允许手动启用。

六家名义免费量不能直接相加。对新注册用户，可持续的月度云端免费量约为：

- 腾讯 500 万 + Azure 200 万 + 阿里 100 万 + 百度 100 万 + Google 50 万 = 950 万字符/月。
- DeepL Developer 的 100 万字符是一次性额度，不是每月额度。

## 对当前仓库的直接影响

当前实现只把帖子 ID 发给 X 接口；第三方厂商都需要帖子原文。因此开发前必须把完整帖子文本安全地传给后台翻译层。

当前成功译文统一持久缓存 7 天。后续缓存至少要保存：

```text
tweetId
sourceTextHash
targetLanguage
provider
translation
createdAt
expiresAt
```

其中：

- 百度结果必须 `cacheable=false`，不得写入 `chrome.storage`、IndexedDB 或其他持久存储。
- Google 结果必须保存 `provider=google`，缓存命中时仍显示 Google 署名。
- 其他厂商目前未发现禁止个人应用本地缓存译文的专项条款，但仍需遵守 X 的内容使用规则。

跨域请求应从扩展后台 Service Worker 发起，并按厂商申请最小范围的 `host_permissions`。Chrome 官方说明，后台 Service Worker 在取得 host permission 后可跨域请求；内容脚本仍受页面同源限制：[Chrome 跨域请求文档](https://developer.chrome.com/docs/extensions/develop/concepts/network-requests)。

## 密钥安全原则

Chrome 官方明确说明扩展本地存储未加密；永久密钥放在客户端始终可能被本机用户、调试工具或恶意软件读取：[Chrome 用户隐私文档](https://developer.chrome.com/docs/extensions/develop/security-privacy/user-privacy)。

个人自用可以接受以下折中，但不等于密钥安全：

- 所有 Key 由用户在设置页手工录入，不写入源码、构建产物、Git、日志或诊断信息。
- 只保存到 `chrome.storage.local`，不使用 `storage.sync`。
- 将 storage access level 限制为 trusted extension contexts，内容脚本不能直接读取密钥。
- 所有签名和第三方请求只在后台 Service Worker 完成。
- 腾讯使用专用 CAM 子用户；阿里使用专用 RAM 用户；Google Key 只允许 Cloud Translation API。
- 提供“测试凭证”“删除凭证”“停用厂商”功能。
- 鉴权失败后禁止无限重试，避免账号风控。
- 如果以后公开发布扩展，永久 Key 直连方案必须改为用户自有后端、短期凭证或本地代理。

## 统一额度与错误状态

后续 Provider 适配层至少统一为以下状态：

| 状态 | 含义 | 行为 |
| --- | --- | --- |
| `quota_exhausted` | 当前免费周期额度已用完 | 禁用到明确的重置时间，然后切下一家 |
| `local_free_cap_reached` | 本地安全阈值已达到 | 在请求发出前停止，避免阿里/百度/Google产生付费 |
| `rate_limited` | 短时频率或突发流量限制 | 指数退避；当前请求可临时切换；冷却后恢复该厂商 |
| `auth_failed` | Key、签名、区域或权限错误 | 禁用厂商并显示配置错误；不要反复重试 |
| `service_not_enabled` | 产品未开通或资源不存在 | 禁用厂商并提供控制台入口 |
| `billing_problem` | 欠费、余额不足或服务隔离 | 禁用厂商并提示检查账单 |
| `unsupported_language` | 当前语向不支持或识别失败 | 只切换当前请求，不永久禁用厂商 |
| `invalid_request` | 参数、长度、编码或程序错误 | 暴露错误并修正/分片，不能靠切下家掩盖 Bug |
| `temporary_failure` | 超时、网络或厂商 5xx | 有限重试，失败后临时切换 |

本地用量记录建议包含：

```text
provider
periodId
estimatedCharacters
reportedCharacters
hardLimit
safetyLimit
blockedUntil
blockedReason
lastSuccessfulCallAt
lastErrorCode
```

厂商未公开重置时区时，不应在代码中猜测。优先以控制台和服务端恢复为准，并允许用户手工“重新启用/重置本地计数”。

### 注册与开通入口

- [注册腾讯云](https://cloud.tencent.com/register)

- [登录腾讯云](https://cloud.tencent.com/login)
- [个人实名认证](https://console.cloud.tencent.com/developer/auth)
- [机器翻译 TMT 控制台](https://console.cloud.tencent.com/tmt)
- [免费资源包与用量](https://console.cloud.tencent.com/tmt/resource_bundle)
- [API 密钥管理](https://console.cloud.tencent.com/cam/capi)
- [官方快速入门](https://cloud.tencent.com/document/product/551/104415)

建议创建专用 CAM 子用户，只授予 `TextTranslate` 权限，不使用主账号永久密钥。保持机器翻译控制台中的后付费关闭。

### 免费规则

- 文本翻译每月 500 万字符。
- 首次开通当天发放，以后每月 1 日发放，仅当月有效。
- 汉字、字母、标点和空格各按一个字符；失败请求不计费。
- 扣减顺序：免费额度 → 预付费资源包 → 后付费。
- 后付费默认关闭；关闭时额度耗尽会停服，不会自动产生费用。

官方来源：[腾讯 TMT 计费概述](https://cloud.tencent.com/document/product/551/35017)。

### API 与限制

- 接口：`TextTranslate`
- API 版本：`2018-03-21`
- Endpoint：`https://tmt.tencentcloudapi.com/`
- 签名：`TC3-HMAC-SHA256`
- 请求头：`Authorization`、`Host`、`Content-Type`、`X-TC-Action`、`X-TC-Version`、`X-TC-Timestamp`、`X-TC-Region`；临时凭证还需 `X-TC-Token`。
- 请求体核心：`SourceText`、`Source=auto`、`Target=zh`、`ProjectId=0`。
- 单次最多 2000 字符。
- `TextTranslate` 默认 5 QPS，维度是 API + 接入地域 + 子账号。

官方来源：[API 概览](https://cloud.tencent.com/document/product/551/15612)、[请求限制](https://cloud.tencent.com/document/product/551/32572)、[公共参数](https://cloud.tencent.com/document/api/551/15615)。

### 错误处理

腾讯 Cloud API 的业务错误可能仍返回 HTTP 200，必须检查 `Response.Error.Code`。

- 免费额度耗尽：`FailedOperation.NoFreeAmount` → 禁用到下月，切下一家。
- 短时限流：`RequestLimitExceeded*`、`LimitExceeded.LimitedAccessFrequency` → 退避并临时切换。
- 鉴权/权限：`AuthFailure.*`、`UnauthorizedOperation` → 禁用并提示检查 Key、签名、CAM 权限。
- 未开通：`FailedOperation.UserNotRegistered` → 禁用并给出 TMT 控制台入口。
- 欠费/停服：`FailedOperation.ServiceIsolate`、`FailedOperation.StopUsing` → 禁用并提示账单。
- 文本过长、语向不支持、识别失败 → 仅切换当前请求或先修正输入。
- `InternalError.*`、`ServiceUnavailable` → 有限重试后临时切换。

官方来源：[腾讯 TMT 错误码](https://cloud.tencent.com/document/api/551/30637)。

### 用量与待验证项

- 响应中的 `UsedAmount` 可用于累计本次实际消耗字符数。
- 没有公开的免费剩余额度查询 API；以本地累计和控制台资源包页交叉核对。
- 官方未承诺浏览器 CORS；取得真实凭证后必须在扩展后台做一次 smoke test。

## 2. Azure Translator F0

### 注册与开通入口

- [注册 Azure 免费账户](https://azure.microsoft.com/free/)
- [Azure Portal](https://portal.azure.com/)
- [创建 Translator 资源指南](https://learn.microsoft.com/en-us/azure/ai-services/translator/how-to/create-translator-resource)
- [价格](https://azure.microsoft.com/en-us/pricing/details/cognitive-services/translator/)
- [服务限制](https://learn.microsoft.com/en-us/azure/ai-services/translator/service-limits)

Portal 中搜索 `Translator`，创建单服务资源，首选 `Region=Global`、`Pricing tier=F0`。创建后在 `Resource Management > Keys and Endpoint` 获取 Key 和 Endpoint。

### 免费规则

- F0 每月 200 万字符，每个 Azure 订阅只能创建一个 F0 Translator 资源。
- F0 还有每小时 200 万字符的滑动窗口，建议均匀使用，约不超过 33,300 字符/分钟。
- F0 月额度耗尽返回 `403001`，不会自动升级为付费层。
- 建议本地 190 万字符预警，但最终可以依赖 `403001` 硬停。

### API 与限制

- 稳定接口：Text Translation REST v3。
- Endpoint：`POST https://api.cognitive.microsofttranslator.com/translate?api-version=3.0&to=zh-Hans`
- Global 单服务鉴权头：`Ocp-Apim-Subscription-Key`。
- 区域型或多服务资源还需 `Ocp-Apim-Subscription-Region`。
- `from` 可省略并自动识别；简中用 `zh-Hans`。
- Translate 最多 1000 个数组元素；单次所有目标语言合计最多 50,000 字符。
- 多目标语言按源字符数乘目标语言数计费。

官方来源：[REST 快速入门](https://learn.microsoft.com/en-us/azure/ai-services/translator/text-translation/quickstart/rest-api)、[Translate v3](https://learn.microsoft.com/en-us/azure/ai-services/translator/text-translation/reference/v3/translate)。

### 错误处理

- `403001`：F0 免费额度耗尽 → 禁用到下月并切换。
- `429000`、`429001`、`429002`：速率/突发限制 → 退避后临时切换，不能记为月额度耗尽。
- `401000`、`401015`：Key 缺失、无效或产品类型错误 → 禁用并提示配置。
- 普通 `403000`：操作不允许 → 检查详情，不能直接当额度耗尽。
- `408001`、`408002`、`500000`、`503000`：有限重试后临时切换。
- `400077` 或其他 400：请求过大/参数错误 → 修正或拆分。

官方来源：[Azure Translator 状态与错误码](https://learn.microsoft.com/en-us/azure/ai-services/translator/text-translation/reference/status-response-codes)。

### 用量、数据与待验证项

- 响应头 `X-metered-usage` 是本次计费字符数。
- Portal Metrics 可查 `TextCharactersTranslated`、`BlockedCalls` 和 `ClientErrors`。
- 没有可用 Translator Key 直接访问的简单剩余额度 API。
- Azure Budget 只告警，不会自动停资源；F0 本身的硬上限才是防扣费保障。
- 文本翻译不持久保存客户文本：[数据、隐私与安全](https://learn.microsoft.com/en-us/azure/ai-foundry/responsible-ai/translator/data-privacy-security)。
- 未发现禁止本地缓存译文或强制署名的规则。
- 月重置具体时区和扩展后台直连能力需用真实账户验证。

账号需实名认证，并在机器翻译控制台单独开通通用版文本翻译。建议创建专用 RAM 用户，只允许 `alimt:TranslateGeneral`。

### 免费规则与计费风险

- 通用版每月 100 万字符，每月 1 日 00:00 更新，不结转。
- 标点、空格、HTML 标签均计入字符。
- 免费额度耗尽后自动进入后付费，并按小时出账。
- 阿里预算、高额消费和余额功能主要用于告警，不提供可靠扣费硬上限。
- 必须在扩展本地约 98 万字符停止调用；不能等待服务端返回免费额度耗尽错误。

官方来源：[计费概述](https://help.aliyun.com/zh/machine-translation/product-overview/billing-overview)、[扣费上限 FAQ](https://help.aliyun.com/zh/user-center/support/billing-faqs)。

### API 与限制

- 产品代码：`alimt`
- 接口：`TranslateGeneral`
- API 版本：`2018-10-12`
- 当前示例 Endpoint：`mt.cn-hangzhou.aliyuncs.com`
- 核心字段：`FormatType=text`、`Scene=general`、`SourceLanguage=auto`、`TargetLanguage=zh`、`SourceText`。
- 通用版 50 QPS，主账号与 RAM 用户合计。
- 官方文档对长度分别写“上限 5000”和“少于 5000”；实现按最多 4999 字符。
- 当前文档仍存在旧 RPC 签名和新 SDK 签名说明冲突；实现时应以 OpenAPI 元数据和当前 V2.0 SDK 生成结果为准，不能混用 ROA 文档。

官方来源：[通用版调用指南](https://help.aliyun.com/zh/machine-translation/developer-reference/api-reference-machine-translation-universal-version-call-guide)、[使用限制](https://help.aliyun.com/zh/machine-translation/developer-reference/limits)。

### 错误处理

- 没有“免费额度耗尽”错误；本地 98 万硬截止才是切换依据。
- `Throttling*`：短时限流 → 退避后临时切换。
- `10009`：RAM 无权限；`10010`：服务未开通 → 禁用并提示配置。
- `10013`：服务未开通或欠费，含义混合 → 禁用并提示检查控制台/账单，不能当免费额度耗尽。
- `InvalidAccessKeyId.*`、`SignatureDoesNotMatch`、`Forbidden*` → 禁用并提示 Key/签名/认证问题。
- `10003`～`10008` 中的编码、参数、语向、识别、长度问题 → 修正输入或仅切换当前请求。
- `10001`、`10002`、`10007`、`10012`、`19999`、`InternalError`、`ServiceUnavailable` → 有限重试后临时切换。

### 用量与待验证项

- 控制台“监控统计”查看调用量和 QPS；费用与成本中的资源实例管理查看资源包余量。
- 没有公开免费额度剩余量 API；`WordCount` 不能当计费剩余额度。
- 官方未承诺 CORS；真实 AK 到位后要验证 Endpoint、签名版本、HTTP/业务错误包装。

新版 `API Key` 页面主要面向大模型翻译 API。通用文本翻译高级版仍应使用控制台提供的 APPID + 密钥，不能混用两套凭证。

### 免费规则与计费风险

- 个人认证后的高级版：每月 100 万字符，QPS 10。
- 超出部分按 49 元/百万字符计费。
- 用量约每 5 分钟更新，费用通常次日结算；免费额度刚耗尽或余额为 0 时不会立即停服。
- 没有可靠的免费额度硬停或公开剩余额度 API。
- 必须在扩展本地约 95 万字符停止调用，并用控制台人工校准。

官方来源：[高级版开通与权益](https://fanyi-api.baidu.com/access/0)。

### API 与限制

- Endpoint：`https://api.fanyi.baidu.com/api/trans/vip/translate`
- 推荐 POST：`application/x-www-form-urlencoded`
- 核心字段：`q`、`from`、`to`、`appid`、`salt`、`sign`
- 签名：`MD5(appid + q + salt + 密钥)`
- `from=auto`，简中目标 `to=zh`。
- 高级版 10 QPS；扩展自身建议限制到 5～8 QPS。
- 旧官方资料常写单次 `q` 不超过 6000 字节，但当前动态文档未稳定展示；开发时先按 5000 UTF-8 字节保守分片，拿到 Key 后实测。

### 错误处理

- `52001`、`52002`：超时/系统错误 → 有限重试后临时切换。
- `52003`：未授权；`54001`：签名错误 → 禁用并提示凭证。
- `54003`：访问频率受限 → 退避，不能记为月额度耗尽。
- `54004`：余额不足 → 禁用到充值或下月并切换。
- `54005`：长 query 频率受限 → 拆短、降速。
- `58000`：客户端 IP 非法；`58002`：服务关闭 → 禁用并提示控制台配置。
- `58001`：语向不支持 → 仅切换当前请求。

### 客户端缓存硬约束

百度官方服务协议明确：如果应用包含客户端程序，客户端程序不能缓存“百度翻译数据”，翻译结果属于百度翻译数据。

因此百度 Provider 必须：

- 设置 `cacheable=false`。
- 不把百度译文写入 `chrome.storage`、IndexedDB、文件或其他持久缓存。
- 只在当前页面 DOM 和必要的短生命周期内存中显示。
- 不向第三方销售、分发、再授权或传送百度翻译数据。
- 如未来希望缓存，必须先取得百度书面许可。

官方来源：[百度旧接入文档、FAQ 与服务协议](https://api.fanyi.baidu.com/api/trans/product/apidoc/)。该页面目前可能跳转，但官方索引仍可检索到上述协议内容。

### 浏览器风险与待验证项

- 百度官方要求不要在浏览器/客户端代码暴露密钥，旧 FAQ 还建议使用固定服务器 IP。
- 普通网页 OPTIONS 检查没有返回允许跨域的 CORS 头；扩展后台加 `host_permissions` 后仍要真实测试。
- 动态文档中的当前单次长度、完整语种表和 HTTP 错误包装需在注册后确认。

## 5. Google Cloud Translation

### 注册与开通入口

- [Google Cloud Console](https://console.cloud.google.com/)
- [创建项目](https://console.cloud.google.com/projectcreate)
- [Billing](https://console.cloud.google.com/billing)
- [启用 Cloud Translation API](https://console.cloud.google.com/apis/library/translate.googleapis.com)
- [创建和管理 API Key](https://console.cloud.google.com/apis/credentials)
- [Translation 配额](https://console.cloud.google.com/apis/api/translate.googleapis.com/quotas)
- [官方初始化文档](https://docs.cloud.google.com/translate/docs/setup)

建议创建独立项目，绑定 Billing，启用 Cloud Translation API，创建标准 API Key，并把 API restriction 限制为 Cloud Translation API。

### Basic v2 与 Advanced v3

个人 Chrome 扩展应选 Basic v2：

- Basic v2 支持 API Key，适合聊天、社交媒体和评论等短文本。
- Advanced v3 需要 OAuth/IAM 或服务账号，不支持普通 API Key。
- 服务账号私钥不能放进扩展；没有后端时不选 v3。

官方来源：[API 版本概览](https://docs.cloud.google.com/translate/docs/api-overview)、[身份验证](https://docs.cloud.google.com/translate/docs/authentication)。

### 免费规则与计费风险

- 标准 NMT 文本翻译和检测前 50 万字符/月免费，但实际实现是 Basic + Advanced 共享的每月最高 10 美元抵扣。
- 必须启用 Billing。
- 超出 50 万字符后自动按约 20 美元/百万字符计费，不会返回“免费额度耗尽”。
- 默认每日字符配额是无限；可以在配额控制台主动降低。
- 必须本地设置 45～48 万字符硬截止，并把控制台每日配额作为第二层保护。
- Billing Budget 默认只告警，不会自动停止 API。

官方来源：[价格](https://cloud.google.com/translate/pricing)、[配额和限制](https://docs.cloud.google.com/translate/quotas)、[预算](https://docs.cloud.google.com/billing/docs/how-to/budgets)。

### API 与限制

- Basic v2 Endpoint：`POST https://translation.googleapis.com/language/translate/v2`
- 建议把 Key 放在 `X-Goog-Api-Key` 请求头，避免进入 URL 日志。
- `q` 最多 128 个字符串，单请求最大 100 KB；官方建议单次约 5000 code points 以内。
- `source` 可省略自动识别；帖子用 `format=text`；简中目标 `zh-CN`。
- 默认通用模型每项目/每用户每分钟 600 万字符；v2 每项目每分钟 300,000 请求。
- API Key 模式的“用户”按客户端 IP 计算。

官方来源：[v2 translate 方法](https://docs.cloud.google.com/translate/docs/reference/rest/v2/translate)。

### 错误处理

- 没有免费月额度耗尽错误；本地硬截止才是切换依据。
- `403 Daily Limit Exceeded`：人为设置的日配额耗尽 → 禁用到太平洋时间午夜。
- `403 User Rate Limit Exceeded`、`429 RESOURCE_EXHAUSTED`：短时限流 → 退避后临时切换。
- 其他 403 可能是 Billing、API 未启用、Key restriction 或权限问题，必须解析详情。
- `API_KEY_INVALID`、`keyInvalid`：禁用并提示重新配置。
- `400 INVALID_ARGUMENT`：修正参数或拆分，不静默掩盖。
- 500/503：有限重试后临时切换。

### 强制署名与缓存

未经人工修改的 Google 译文直接展示给用户时，Google 要求：

- 明确告知用户正在查看 Google Translate 自动翻译。
- 在每个译文旁显示 `powered by Google Translate` 官方署名。
- 应用说明和帮助文档写明使用 Google Translate，并链接 Google Translate。
- 在应用内提供官方免责声明。
- 缓存命中时仍必须显示署名。

所以缓存不能只存纯文本，必须保存 `provider=google`；渲染层也必须保留来源展示能力。

官方来源：[Google Translation 署名要求](https://docs.cloud.google.com/translate/attribution)、[HTML 标记要求](https://docs.cloud.google.com/translate/markup)。

### 数据、安全与待验证项

- Google 声明输入文本只在内存中短暂处理，不用于其他目的：[数据使用 FAQ](https://docs.cloud.google.com/translate/data-usage)。
- 未发现禁止普通应用本地缓存译文的现行专项条款。
- Google 不建议在客户端放 API Key；个人自用仍需接受本机可提取的风险。
- `chrome-extension://` 来源能否用于应用限制、扩展后台直连/CORS、月抵扣重置时区需要真实 Key 验证。

## 6. DeepL

### 注册与账户入口

- [API 注册/套餐选择](https://www.deepl.com/en/checkout?is_api=true)
- [登录 DeepL](https://www.deepl.com/login)
- [账户概览](https://www.deepl.com/account/summary)
- [API Key 管理说明](https://support.deepl.com/hc/en-us/articles/360020695820-API-key-for-DeepL-API)
- [最新 API 套餐说明](https://support.deepl.com/hc/en-us/articles/360021200939-DeepL-API-plans)

登录后进入 `Account > API Keys & Limits` 创建 Key。

### 2026 年免费计划变化

- `DeepL API Free` 已停止新购。旧账号仍保留每月 50 万字符。
- 新用户对应 `DeepL API Developer`：总计 100 万字符，额度耗尽后不会重置，只能升级。
- 注册结账页仍可能显示“免费翻译 50 万字符”，与最新帮助中心的 Developer 规则不一致。
- 注册后必须以实际套餐名和 `/v2/usage` 返回的 `character_limit` 为准。

因此 DeepL 对新用户不是可持续的月度免费池。

### API 与限制

- 主版本：v2。
- 翻译：`POST /v2/translate`
- 旧 API Free 基址：`https://api-free.deepl.com`
- 付费基址：`https://api.deepl.com`
- 鉴权：`Authorization: DeepL-Auth-Key <key>`
- `text` 最多 50 项；总请求体最大 128 KiB；Header 最大 16 KiB。
- `source_lang` 可省略；简中目标建议 `ZH-HANS`。
- 用量：`GET /v2/usage`，返回 `character_count`、`character_limit`。
- 新 Developer 的公开文档尚未明确更新基址和 Key 后缀，注册后必须按账户 Quickstart 验证，代码不能猜。

官方来源：[翻译 API](https://developers.deepl.com/api-reference/translate)、[用量与限制](https://developers.deepl.com/docs/resources/usage-limits)、[用量查询](https://developers.deepl.com/api-reference/usage-and-quota)。

### 错误处理

- HTTP `456`：额度/成本控制上限 → 立即切换。旧 Free 禁用到下个周期；Developer 达一次性总额度后保持禁用直到升级。
- `429`：动态限流 → 指数退避和抖动，失败后临时切换。
- `403`：鉴权/订阅问题 → 禁用并提示配置。
- `400`、`404`、`413`、`414`：参数、URL 或请求大小问题 → 修正，不盲目重试。
- 5xx/503：有限重试后临时切换。

官方来源：[DeepL 错误处理](https://developers.deepl.com/docs/best-practices/error-handling)。

### 阻断级约束

DeepL 官方明确不允许浏览器应用直接调用 API，并要求 API Key 不得放在客户端；推荐通过自己的后端代理：[浏览器 CORS 与代理说明](https://developers.deepl.com/docs/best-practices/cors-requests)。

更重要的是，最新 DeepL API Developer 条款允许永久保存 Content/Processed Content，并规定 Developer API 不得处理个人数据。X 帖子经常包含姓名、用户名、`@handle` 或其他个人信息，因此 Developer 免费档不适合无差别自动翻译 X 帖子。

官方来源：[DeepL Pro/API 条款](https://www.deepl.com/en/pro-license.html)。

旧 API Free 用户也不能把 Key 直接放入扩展正式调用；至少需要自有本地/远端代理。若以后接入 DeepL，还需要按条款核对终端用户展示时的 DeepL 品牌要求。

### 接入决定与待验证项

- 新用户默认关闭 DeepL，不计入月度轮换池。
- 已有 legacy API Free 用户可以保留配置入口，但只有代理可用时才启用。
- 注册后需验证新 Developer 的实际基址、Key 类型、`/usage` 响应和中国大陆注册可用性。

## 拿到 Key 后的最小验证清单

每家都应先在开发环境完成以下 smoke test，再开始写完整轮换逻辑：

1. 翻译英文、日文各一条短帖到简体中文。
2. 验证自动语种识别、emoji、链接、`@handle` 和 hashtag。
3. 记录真实成功响应、字符计量字段和响应头。
4. 使用故意错误的 Key，确认鉴权错误包装。
5. 发送超长文本，确认长度错误和分片边界。
6. 低速连续请求，确认 Service Worker 跨域、CORS 和 host permission。
7. 控制并发触发限流，确认 429/业务限流不会被误判为月额度耗尽。
8. 对有用量接口的厂商核对本地计数和服务端计数。
9. 确认阿里、百度、Google 在本地安全阈值后不再发请求。
10. 确认百度译文不会进入持久缓存，Google 缓存译文仍显示署名。

## 后续开发顺序建议

1. 先实现统一 Provider 接口、错误分类和本地额度账本。
2. 接入腾讯，验证 `UsedAmount`、业务错误 HTTP 200 和 `NoFreeAmount`。

8. DeepL 暂缓，除非后续明确增加代理并重新审查 Developer 数据条款。
